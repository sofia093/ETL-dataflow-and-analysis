# ETL-dataflow-and-analysis
This project implements a complete Data Warehouse solution using Oracle technologies. 

## ETLdataflow
This document presents the complete ETL (Extract, Transform, Load) implementation used to populate the data warehouse dimensions and fact table. It contains screenshots and execution results from Oracle Data Transforms and SQL Developer.
### 1. Product Dimension (DIM_PRODUS)
Shows the ETL mapping created for loading data into the DIM_PRODUS dimension.
### 2. Supplier Dimension (DIM_FURNIZOR)
Presents the ETL mapping responsible for loading supplier data.
### 3. Warehouse Dimension (DIM_DEPOZIT)
Documents the data flow used to populate warehouse/deposit information.
### 4. Time Dimension (DIM_TIMP)
Contains the ETL process used to generate and load the time dimension.
### 5. Inventory Fact Table (FACT_INVENTAR)
Shows the ETL workflow used to populate the central fact table.

## DataAnalysis
This document presents the analytical layer built on top of the data warehoouse. Documents the creation of the Oracle Analytic View used for multidimensional analysis. Presents the dashboard created using the analytical model. Shows how end users can explore inventory and business data through interactive reports and charts.
